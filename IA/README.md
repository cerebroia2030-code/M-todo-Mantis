## 🎯 Visión General

El Sistema de Asistentes Mantis automatiza y estandariza el ciclo completo de consultoría:

```
VENTAS → DIAGNÓSTICO → ESTRATEGA → PROPUESTA → ARQUITECTO → AUTOMATIZADOR
      ↑                                                             ↓
      └─────────────── FEEDBACK & MEJORA CONTINUA ──────────────────┘
```

**Objetivo**: Transformar clientes desde el primer contacto hasta valor medible en 90 días.

## 🤖 Asistentes Disponibles

### 1. 🎯 **Ventas-Mantis.md**
**Especialidad**: Prospectación y Cierre de Deals

| Aspecto | Descripción |
|--------|-------------|
| **¿Qué hace?** | Guía prospectos desde first contact hasta diagnosis |
| **¿Cuándo se usa?** | Fase 0: Cuando tienes un lead o empresa objetivo |
| **¿Qué recibe?** | Lead profile, buyer personas, histórico de ventas |
| **¿Qué entrega?** | Pitch personalizado, scripts, objection handling, playbook |
| **Responsable** | Sales Manager / Sales Rep |
| **Duración** | 1-4 semanas (según ciclo ventas) |

**Flujo específico:**
```
Lead → Warm-up → First contact → Qualification → 
Discovery call → Schedule diagnosis → ✅ Handoff a Diagnostico
```

**Outputs principales:**
- 📊 Pitch deck personalizado
- 💬 Script de prospectación
- 🚨 Objection handling guide
- 📅 Follow-up cadence
- 📖 Sales playbook

---

### 2. 🔍 **Diagnostico-Mantis.md**
**Especialidad**: Auditoría Empresarial y Oportunidades

| Aspecto | Descripción |
|--------|-------------|
| **¿Qué hace?** | Analiza profundamente procesos, detecta automatizaciones e IA |
| **¿Cuándo se usa?** | Fase 1: Una vez el cliente está calificado e interesado |
| **¿Qué recibe?** | Contexto empresa, procesos, datos, presupuesto |
| **¿Qué entrega?** | Reporte diagnóstico con ROI, roadmap 90 días, stack tech |
| **Responsable** | Consultor Senior / Auditor |
| **Duración** | 2-3 semanas |

**Flujo específico:**
```
Descubrimiento (4 fases) → Análisis de procesos → 
Identificación de oportunidades → Matriz impacto-esfuerzo → 
Roadmap 30-60-90 → Reporte ejecutivo → ✅ Handoff a Propuesta
```

**Outputs principales:**
- 📋 Reporte diagnóstico (12 secciones)
- 📊 Matriz impacto vs esfuerzo
- 📈 Roadmap implementación 90 días
- 💰 Stack tecnológico recomendado (3 opciones)
- 🎯 ROI proyectado

---

### 3. 📋 **Propuesta-Mantis.md**
**Especialidad**: Comercialización y Cierre

| Aspecto | Descripción |
|--------|-------------|
| **¿Qué hace?** | Convierte diagnóstico en propuesta ganadora |
| **¿Cuándo se usa?** | Fase 2: Después de diagnosis, antes de contratación |
| **¿Qué recibe?** | Reporte diagnóstico, contexto presupuestario, stakeholders |
| **¿Qué entrega?** | Propuesta PDF, modelo precios, términos, FAQ |
| **Responsable** | Sales Manager / Account Manager |
| **Duración** | 1-2 semanas |

**Flujo específico:**
```
Análisis comercial → Estructura de propuesta → 
Opciones de servicio (Básico/Estándar/Premium) → 
ROI financiero → Presentación → ✅ Cierre o feedback
```

**Outputs principales:**
- 📄 Propuesta ejecutiva (5-8 págs)
- 💰 Desglose de precios
- 💹 Modelo financiero 3 años
- 📊 Presentación ejecutiva + slides
- 📋 Términos y condiciones
- ❓ FAQ objeciones

**Si se GANA**: → Envía a Arquitecto-Mantis
**Si se PIERDE**: → Feedback a Ventas-Mantis para mejorar

---

### 4. 🏗️ **Arquitecto-Mantis.md**
**Especialidad**: Diseño Técnico e Ingeniería

| Aspecto | Descripción |
|--------|-------------|
| **¿Qué hace?** | Diseña blueprint técnico de la solución |
| **¿Cuándo se usa?** | Fase 3: Una vez propuesta ganada |
| **¿Qué recibe?** | Propuesta, sistemas cliente, restricciones técnicas |
| **¿Qué entrega?** | Arquitectura, specs técnicas, roadmap sprints |
| **Responsable** | Tech Lead / Solution Architect |
| **Duración** | 1-2 semanas |

**Flujo específico:**
```
Descubrimiento técnico → Diseño arquitectura → 
Especificaciones detalladas → Modelo de datos → 
Integración plan → Sprint roadmap → ✅ Handoff a Automatizador
```

**Outputs principales:**
- 🏗️ Diagrama de arquitectura (Visio/Miro)
- 📋 Especificación técnica detallada
- 🗄️ Modelo de datos (ER diagram)
- 🔌 Plan de integraciones
- 👥 Estructura de equipo
- 🛡️ Diseño de seguridad
- 📊 Jira roadmap (sprints)

---

### 5. ⚙️ **Automatizador-Mantis.md**
**Especialidad**: Desarrollo e Implementación

| Aspecto | Descripción |
|--------|-------------|
| **¿Qué hace?** | Construye la solución según blueprint |
| **¿Cuándo se usa?** | Fase 4: Post-arquitectura, durante implementación |
| **¿Qué recibe?** | Blueprint técnico, acceso a sistemas, datos de prueba |
| **¿Qué entrega?** | Código, integraciones, ambiente productivo |
| **Responsable** | Dev Team / DevOps / Tech Lead |
| **Duración** | 8-12 semanas (sprints de 2 semanas) |

**Flujo específico:**
```
Sprint planning → Desarrollo (2 semanas) → 
Testing & QA → Integration → UAT → 
Deployment PROD → Cada sprint genera valor incremental
```

**Outputs principales:**
- 💻 Código fuente (Git repo)
- 🔧 Configuraciones (IaC)
- 🔌 Integraciones funcionales
- 🤖 Modelos IA (si aplica)
- 🗄️ Base de datos poblada
- 📚 Documentación técnica
- ✅ Test reports
- 🚀 Ambiente PROD funcional

## 🔄 Flujo Completo de Consultoría

### Fase 1: PROSPECTACIÓN (0-4 semanas)
```
┌─────────────────────────────────────┐
│ VENTAS-MANTIS                       │
│ • Identificar leads                  │
│ • Warm-up (email/social)             │
│ • First contact + demo               │
│ • Calificación                       │
│ • Discovery call                     │
└─────────────────────────────────────┘
           ↓
       ¿Fit?
      /     \
    SÍ       NO
   /           \
  ↓             └→ Guardar para después
```

**Criterio de éxito**: Lead calificado con interés en diagnóstico

---

### Fase 2: DIAGNÓSTICO (2-3 semanas)
```
┌─────────────────────────────────────┐
│ DIAGNOSTICO-MANTIS                  │
│ • 4 fases de análisis profundo      │
│ • Identificar cuellos de botella    │
│ • Quick wins + Strategic projects    │
│ • Roadmap 30-60-90                  │
│ • Stack tecnológico 3 opciones       │
└─────────────────────────────────────┘
           ↓
    Reporte ejecutivo
           ↓
```

**Criterio de éxito**: ROI claro, >70% probabilidad de cierre

---

### Fase 3: PROPUESTA (1-2 semanas)
```
┌─────────────────────────────────────┐
│ PROPUESTA-MANTIS                    │
│ • Cuantificar valor                 │
│ • Estructurar opciones de servicio   │
│ • Presentación ejecutiva             │
│ • Términos y condiciones             │
│ • FAQ objections                     │
└─────────────────────────────────────┘
           ↓
       Presentación
           ↓
        ¿Se gana?
       /         \
      SÍ          NO
     /              \
    ↓                └→ Feedback a Ventas
```

**Criterio de éxito**: Deal cerrado / Contrato firmado

---

### Fase 4: DISEÑO (1-2 semanas)
```
┌─────────────────────────────────────┐
│ ARQUITECTO-MANTIS                   │
│ • Blueprint técnico completo         │
│ • Especificaciones detalladas        │
│ • Integraciones definidas            │
│ • Roadmap sprints                    │
│ • Equipo y roles                     │
└─────────────────────────────────────┘
           ↓
    Blueprint ejecutable
           ↓
```

**Criterio de éxito**: Todo documentado, cliente aprueba, dev team listo

---

### Fase 5: IMPLEMENTACIÓN (8-12 semanas)
```
┌─────────────────────────────────────┐
│ AUTOMATIZADOR-MANTIS                │
│ • Sprint planning semanal             │
│ • Desarrollo (2 semanas/sprint)      │
│ • Testing continuo                   │
│ • UAT con cliente                    │
│ • Deployment incremental             │
│ • Documentación + Runbooks           │
└─────────────────────────────────────┘
           ↓
    Valor entregado incrementalmente
    (Sprint 1, 2, 3...)
           ↓
       Cliente recibe ROI
```

**Criterio de éxito**: ✅ Todas las fases completadas, ROI logrado

3. Usa: Scripts personalizados

1. Abre: Diagnostico-Mantis.md

3. Usa: Preguntas de descubrimiento
4. Entrega: Reporte diagnóstico
5. Objetivo: Identificar ROI
```

### Escenario 3: Ganamos una Propuesta
```
1. Abre: Arquitecto-Mantis.md
2. Sigue: Descubrimiento técnico
3. Entrega: Blueprint completo
4. Abre: Automatizador-Mantis.md
5. Inicia: Sprints de implementación
```

### Escenario 4: Implementación en Curso
```
1. Abre: Automatizador-Mantis.md
2. Sigue: Sprint planning semanal
3. Ejecuta: User stories del sprint
4. Entrega: Incremento de valor c/2 semanas

## 📈 Métricas de Éxito por Fase

| Fase | Métrica | Objetivo | Medida |
|------|---------|----------|--------|
| **Ventas** | Tasa de conversión | >30% leads → diagnosis | # diagnosticos / # leads |
| **Diagnostico** | Tasa de propuesta | >80% diagnosis → propuesta | # propuestas / # diagnosticos |
| **Propuesta** | Win rate | >70% | # contratos / # propuestas |
| **Arquitecto** | Aprobación blueprint | 100% | Cliente aprueba diseño |
| **Automatizador** | ROI logrado | >1.5x Año 1 | $ beneficios / $ inversión |
| **End-to-end** | Ciclo completo | <5 meses | Lead → PROD Live |
